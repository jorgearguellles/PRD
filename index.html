¿Qué es un PRD?

El PRD (Product Requirements Document) es el documento que define qué vas a construir, para quién y por qué, antes de escribir una sola línea de código.

Es importante porque pone a todo el equipo (ahora también a los agentes de IA) de acuerdo sobre el mismo objetivo y evita construir cosas no planeadas.

A diferencia de un PRD tradicional, que está pensado para que un humano implemente el producto, uno para IA está pensado para que un agente pueda razonar, decidir y generar código consistente durante semanas.

No es una especificación.
Es el contexto permanente del proyecto.

El objetivo de un PRD no es describir cada pantalla ni cada botón, sino darle al agente suficiente contexto para tomar decisiones correctas incluso cuando el usuario no está presente. La implementación, la arquitectura y la UI pueden evolucionar — el PRD debe ser estable en el tiempo.
